# Convert Use Cases to Jira Stories

Use this skill to decompose approved use cases into delivery-ready story drafts.

## When to Use This Skill

- Turning use cases into backlog items for implementation planning.
- Splitting a large feature into thin vertical slices with clear dependencies.
- Preparing Jira story drafts before creating the issues.

## Workflow

1. Start from validated use cases, not from raw ideas, when possible.
2. Group work into stories that deliver a coherent slice of value.
3. Order stories by dependency and implementation sequence.
4. For each story, provide:
   - Summary
   - User story statement
   - Scope
   - Acceptance criteria
   - Dependencies
   - Risks or open questions
5. If the user asks to create the issues in Jira, hand off each finalized draft to the Jira workflow already available in this repository.

## Output Rules

- Prefer multiple small stories over one oversized story.
- Keep acceptance criteria behavioral and independently verifiable.
- Separate enablers or technical foundation stories from user-facing stories when that improves delivery clarity.
- Do not create or assume issue keys unless the user explicitly provides them.
