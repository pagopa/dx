Add Azure Managed Redis support via new `managed_redis` input variable.

The `managed_redis` input accepts a list of objects with `id` (AMR resource ID)
and `role` (reader, writer, or owner). Role mapping:

- `reader` → Azure Managed Redis Reader (control-plane read-only)
- `writer` → data-plane `default` access policy (Redis commands)
- `owner`  → Azure Managed Redis Contributor (control-plane) + data-plane access

Resource IDs are validated using `provider::azurerm::parse_resource_id`.

Example:
```hcl
managed_redis = [
  {
    id   = azurerm_managed_redis.cache.id
    role = "writer"
  },
]
```

Legacy `redis` input is unchanged (backward compatible).
